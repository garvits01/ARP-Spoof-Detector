# Contributing to ARP Spoof Detector 🛡️

First off, thank you for considering contributing to this project! It's people like you that make open-source networking and security tools such a great community to be a part of.

### 1. Where do I go from here?

If you've noticed a bug or have a flawless feature idea, feel free to open a specific **Issue** outlining your thoughts. If you'd like to dive straight into the code:
1. Fork the repository natively.
2. Clone your fork to your local machine.

### 2. Setting up your environment

Setting up a Python virtual environment is highly recommended so dependencies do not conflict:
```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### 3. Making Changes

- Ensure your code adheres to standard Python styling (PEP 8).
- Briefly test your changes using a Virtual Machine testing environment to ensure no core spoofing monitoring triggers have been inadvertently disabled.
- Commit your changes with clear, descriptive commit messages.

### 4. Submitting a Pull Request (PR)

- Push your local changes back to your fork.
- Submit a Pull Request from your fork's branch towards our `main` branch.
- In the PR description, explicitly describe what you altered and ideally provide a screenshot or log snipped detailing that it passes tests.

*By contributing, you agree that your contributions will be licensed under its MIT License.*
